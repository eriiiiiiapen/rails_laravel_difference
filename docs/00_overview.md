# RailsとLaravelの違いに関する所感

## Rails

コード量の少なさ。全部入り。
開発者が「何をしたいか」を察して、先回りしてくれる。

＜暗黙感＞
用意されているモデルはuseしなくてもControllerで使用できるし、
viewをreturnせずともルールに従って勝手にviewやJSONを返す。
「名前さえ合っていれば、Railsが裏で繋いでおくよ」 という思想がわかりやすい。

＜万能感＞
ActiveRecordやActiveStorageの万能性と、Hotwireについても基本viewだけで済む楽さ。
「Rails公式が正解を用意しているから、迷わずこれを使え」というフルパッケージ感が、開発スピードを爆速にしてくれる。

## Laravel

コード量は多いが、堅牢さと初学者に優しい。
開発者が「今何を扱っているか」を常に意識させることで、予測可能性を高めてくれる。

＜明示的＞
use 文によって依存関係が見える化され、IDEの補完も効きやすい。

＜状態の分離＞
例えば「クエリを組み立てている最中（QueryBuilder）」と「データを取得した後（Collection）」が明確に分かれているため、
チーム開発でも「今、この変数は何ができるのか」を誤解しにくい。

＜コンポーネント＞
Livewireは、バックエンドのロジックをフロントエンドのコンポーネントとして切り出す感覚があり、
モダンなJSフレームワークに近い秩序を感じられる。
